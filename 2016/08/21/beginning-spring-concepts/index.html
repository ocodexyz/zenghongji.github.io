Spring 基本概念

种一棵树最好的时间是十年前，而后是现在。

ACID特性

  • Atomicity 原子性：要么全部完成，要么全部不完成。
  • Consistency 一致性：数据完整性。
  • Isolation 隔离性：允许并发。
  • Durability 持久性：事务完成后，对数据修改永久不丢失。

微服务
一种以业务功能为主的服务设计概念，每一个服务都具有自主运行的业务功能，对外开放不受语言限制的 API (最常用的是 HTTP)，应用程序则是由一个或多个微服务组成。

POJO Plain Old Java Object。不需要依赖任何环境特定的类或借口就可以推断Java类，并且不需要任何特定的环境就可以运行。

中间件服务 每个应用程序都会出现的独立于业务需求的需求，比如事务、持久化、安全性、远程处理、线程、连接和资源池、缓存、验证和聚类分析等。

容器 表示一种环境，在该环境中可以创建组件并将其捆绑在一起，以及使用容器提供的中间件服务。

IoC Inversion Of Control。控制反转。一种管理组件依赖项的控制模式。在该模式中依赖管理不是由组件自身完成，而是由环境完成，即容器来完成。Don't call us, we'll call you.好莱坞原则。

  • Dependency lookup 依赖查找。容器提供回调方法并在特定阶段由容器调用，并通过查找上下文帮助组件查找其依赖项。
  • Dependency injection 依赖注入。组件定义各自的依赖项，并且在组件创建期间由容器将这些依赖项连接起来。
    1. Setter 注入
    2. 构造函数注入

AOP Aspect-Oriented Programming。面向方面／切面编程。举例：对于一个信用卡应用程序来说，存款、取款、帐单管理是它的主关注点，日志和持久化将成为横切整个对象结构的横切关注点。
